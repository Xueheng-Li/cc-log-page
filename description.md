## CC LOG

CC LOG主要是解决Claude Code的历史和实时日志查看问题。

Claude Code 非常的好用，但是作为命令行工具有一个非常大的问题，就是它的日志不太方便查阅。

CC LOG解决的第一个解决的痛点就是将过往的历史文本记录用图形化更加友好的交互界面进行展示。

CC LOG启动后自动地读取本地的Claude Code的日志存储文件JSONL，将整体的文本内容清晰地划分为项目、会话以及详情三栏，通过切换项目以及不同的会话来方便地查看过往的对话历史记录，全局的掌控Claude的工作流程。

另外的图形化的界面还有效解决了客户端会话截断的问题。

比如我就习惯在iTerm2上使用Claude Code，iTerm2随着输出历史的不断增加，会挤掉之前的记录，只保留临近的会话。

并且历史记录一旦多了之后，命令行窗口滚动查找就特别麻烦，还会经常出现输出跳动的问题。

CC LOG直接包含了完整的历史会话详情的内容。在面板中自动收集了过往所有的会话历史，切换不同项目就可以看到会话里面多个会话记录，在详情面板我们可以看到当前会话之前的所有的输入输出，包含了AI回复和工具调用等等具体的内容。

CC LOG还在在原来的基础上扩展了实时日志，和Claude Code客户端完全同步。

在会话条目面板上，CC LOG可以直接复制会话ID的命令。

反正我每次命令行resume时，看到那一长串的历史列表，常常是一脸懵逼，完全回忆不起来这段会话里面的主要内容。

有了图形化界面之后，我们就有了一个全局化的视角，可以更直观的查看这段会话具体讲了什么内容。

单个会话条目也可以一键下载导出会话的所有内容，有了这些内容之后，可以投喂到大模型用进行总结，将过程提炼汇总成经验。

CC LOG还提供了对话级别的搜索过程，过去的任意片段都在索引完成都可以直接快速搜索直达。
